# 我的课程页功能与信息展示优化

## 目标
- 升级教师端 `我的课程` 页面的信息密度、操作清晰度和详情承载能力。
- 保持现有业务主体可用，在必要处补充前后端接口、权限校验和教师数据隔离。

## 阶段
- [complete] 1. 盘点现有课程前后端能力与缺口
- [complete] 2. 设计课程列表页信息架构、筛选与批量操作
- [complete] 3. 实现课程详情页及快捷入口
- [complete] 4. 补充课程新增编辑字段、封面上传与状态联动
- [complete] 5. 加固教师权限与操作反馈
- [complete] 6. 诊断、构建与交付说明

## 已知约束
- 用户要求的不只是 UI，需要补全前后端功能。
- 教师端课程能力必须只作用于本人课程。
- 需要尽量复用现有课程、作业、学生、OSS 上传能力。

## 风险
- 现有课程实体/DTO 可能缺少开课时间、结课时间、学分、课时、封面等字段。
- 课程详情页可能没有现成聚合接口，需要新增后端查询能力。
- 批量结课/删除会牵涉权限校验与二次确认。

## 已完成补充
- 学生端和教师端个人中心头像上传体验已完成收口。
- 实际数据库头像字段位于统一用户表 `sys_user.avatar_url`，未新增不存在的 `sys_student` / `sys_teacher` 表字段。

## Errors Encountered
| Error | Attempt | Resolution |
|-------|---------|------------|
| 前端构建命令返回 `terminal is disposed` | 1 | 更换新终端重新执行，属于终端会话异常，非代码错误 |

## 2026-05-04 成绩页班级切换自动联动作业与成绩

### 目标
- 教师端成绩页只选择班级后，自动加载该班级作业列表、默认选中第一项作业，并立即渲染成绩表格与统计数据。
- 后端补充 `GET /assignment/listByClass/{classId}` 能力，按创建时间倒序返回该班级作业列表。

### 阶段
- [complete] 1. 盘点 `GradeManage.vue`、前端作业 API 与后端作业控制器现状
- [complete] 2. 修改前端班级切换逻辑，复用原有成绩查询方法
- [complete] 3. 补充后端按班级查询作业列表接口
- [complete] 4. 诊断修改文件并做基本验证

### 已知约束
- 用户要求保留原有成绩查询方法主体逻辑，不重构页面结构。
- 前端技术栈为 Vue2 风格交互 + Element UI，但当前项目文件实际使用 Vue `script setup` 形式，需要在现有写法内完成兼容修改。

### 风险
- 页面当前变量名使用 `filters.assignmentId`，与用户口述的 `queryParams.homeworkId` 不一致，需要按现有代码最小改动处理。
- 路由预置 `classId/assignmentId` 的初始化逻辑需要和“默认选中第一个作业”兼容，避免覆盖显式传入的作业参数。

### 结果
- 前端已在班级切换时执行“清空旧数据 -> 拉取班级作业 -> 默认选中首个作业 -> 自动查询成绩”。
- 前端新增 `getHomeworkListByClassId(classId)`，调用新接口 `/homework/listByClass/{classId}`。
- 后端已新增 `HomeworkController` 并按 `createdAt desc, id desc` 返回当前教师可访问的班级作业。
- 验证通过：`frontend` 执行 `npm run build` 成功，`backend` 执行 `mvn compile -DskipTests` 成功。

## 2026-05-04 我的课程封面图不显示修复

### 目标
- 修复教师端/学生端“我的课程”课程卡片封面图不显示，只剩默认灰色占位的问题。
- 兼容历史相对路径封面地址，并为课程卡片补充懒加载与失败兜底。

### 阶段
- [complete] 1. 排查课程封面字段、前端绑定方式与后端静态资源访问链路
- [complete] 2. 修复前端课程封面 URL 解析与渲染方式
- [complete] 3. 修复后端课程封面历史路径兼容
- [complete] 4. 诊断与构建验证

### 已知约束
- 当前项目虽然按 Vue2 + Element UI 的交互风格实现，但实际代码是 Vue `script setup` + Element Plus，需要在现有结构内最小改动。

- 后端 `CourseServiceImpl` 已在课程保存/查询时统一规范 `coverUrl`，避免旧数据相对路径继续导致图片请求 404。
- 验证通过：`frontend` 执行 `npm run build` 成功，`backend` 执行 `mvn compile -DskipTests` 成功。
