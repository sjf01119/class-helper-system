# 学习辅助系统项目文档

## 1. 文档说明

本文档基于以下两部分信息编写：

- 毕业设计论文《基于 Java Web 的“智健优选”线上购药系统的设计与实现》中的工程方法论（需求分析、角色建模、模块化设计、分层实现、部署与测试流程）。
- 当前仓库 `class-helper-system` 的真实代码实现（教学场景，非医药场景）。

说明：本项目在业务领域上已从“医药电商”切换为“校园学习辅助”，但沿用了论文中的系统化开发思路。

## 2. 项目概述

- 项目名称：学习辅助系统（Class Helper System）
- 项目目标：为管理员、教师、学生提供统一的教学协作平台，覆盖课程、班级、作业、公告、资料、成绩与日志管理。
- 架构形态：前后端分离，REST API 驱动。
- 核心价值：
- 统一教学数据入口，降低沟通与管理成本。
- 建立角色化权限体系，保障数据边界与操作安全。
- 支持作业全过程（发布-提交-批改-查询）闭环。

## 3. 技术架构

### 3.1 总体架构

- 前端：Vue 3 + TypeScript + Vite + Element Plus + Pinia + Vue Router
- 后端：Spring Boot 3.2.2 + Spring Security + JWT + MyBatis-Plus
- 数据层：MySQL 8.x
- 缓存层：Redis（已引入依赖，可用于会话/热点数据扩展）
- 文件存储：本地上传目录 + 阿里云 OSS（已集成 OSS SDK）

### 3.2 关键工程特性

- 认证鉴权：基于 JWT 的登录态管理与角色访问控制。
- 权限控制：`@RequiresRole` 注解 + 过滤器/切面协同。
- 数据访问：MyBatis-Plus + Mapper 分层组织。
- 统一响应：`R`、`PageResult` 等统一返回结构。
- 日志能力：登录日志、操作日志、AOP 记录关键行为。

## 4. 角色与权限

系统当前角色为三类：

- 管理员（admin）：用户管理、班级管理、课程管理、公告管理、系统看板、日志查看。
- 教师（teacher）：我的班级、我的课程、作业管理、学习资料、公告管理、成绩管理、个人中心。
- 学生（student）：我的班级、我的课程、作业中心、学习资料、公告通知、成绩查询、个人中心。

权限原则：

- 页面层按路由 `meta.roles` 控制访问。
- 接口层按角色注解与安全上下文双重校验。
- 数据层按“本人/本班/本课程”范围收敛查询。

## 5. 功能模块

### 5.1 管理端

- 用户管理：教师/学生账号的新增、编辑、重置密码、状态维护。
- 班级管理：班级信息维护、邀请码重置、班级成员关系管理。
- 课程管理：课程创建与维护、课程状态调整、按班级分配课程。
- 公告管理：系统公告发布、分页查询、批量删除与详情管理。
- 日志审计：登录日志、操作日志查询与追踪。
- 数据看板：管理员首页统计（用户增长、班级分布、公告概览等）。

### 5.2 教师端

- 我的班级：查看所带班级、班级学生、邀请码信息。
- 我的课程：课程列表与详情、教学信息维护。
- 作业管理：发布作业、上传附件、发布/撤回、查看提交统计、批改作业。
- 学习资料：资料上传、下载、预览、可见性控制。
- 公告管理：面向班级或课程范围发布通知。
- 成绩管理：对提交结果进行评分并反馈。

### 5.3 学生端

- 班级加入：通过邀请码加入班级。
- 我的课程：查看可学习课程和课程详情。
- 作业中心：查看待办作业、提交作业、查看批改结果。
- 学习资料：按课程查看资料并下载/预览。
- 公告通知：查看系统与课程公告。
- 成绩查询：查看课程/作业成绩结果。

## 6. 代码结构（核心）

```text

├─ backend/
│  ├─ src/main/java/com/example/classhelper/
│  │  ├─ controller/     # 接口层

│  │  ├─ config/         # 安全、跨域、MyBatis、OSS 等配置
│  │  ├─ filter/aspect/  # JWT 过滤、权限切面、操作日志切面
│  │  └─ utils/          # JWT/Security 工具
│  └─ src/main/resources/application.yml
├─ frontend/
│  ├─ src/views/         # 管理员/教师/学生页面
│  ├─ src/router/        # 路由与鉴权守卫
│  ├─ src/stores/        # 用户状态管理
│  └─ src/api/           # 接口请求封装
└─ docs/
   ├─ backend/sql/       # 初始化与增量 SQL
   └─ deploy/            # 部署文档与脚本
```

## 7. 运行与部署

### 7.1 本地开发

1. 准备环境：JDK 17、MySQL 8、Node.js 18+。
2. 创建数据库：`class_helper_db`，执行 `docs/backend/sql/` 中初始化/升级脚本。
3. 启动后端：
```bash
cd backend
mvn spring-boot:run
```
4. 启动前端：
```bash
cd frontend
npm install
npm run dev
```
5. 访问地址：
- 前端：`http://localhost:5173`
- 后端：`http://localhost:8080/api`

### 7.2 生产部署建议

- 前端：`npm run build` 后由 Nginx 托管 `dist/`，并配置 SPA 路由回退。
- 后端：`mvn clean package -DskipTests` 生成 Jar，使用 `prod` 配置运行。
- 反向代理：Nginx 转发 `/api` 至后端服务。
- 配置项：数据库、CORS、JWT、OSS 建议通过环境变量管理。

## 8. 测试与验收

### 8.1 接口测试

- 参考 `backend/API-TEST.md` 进行登录、鉴权与角色接口验证。
- 默认测试角色包含管理员、教师、学生账号。

### 8.2 关键验收点

- 认证流程：登录成功后不同角色进入各自工作台。
- 权限隔离：角色越权访问被正确拦截。
- 教学闭环：课程发布、作业提交与批改链路可完整执行。
- 日志可追溯：登录日志和操作日志可查询。
- 文件可用性：资料上传、下载、预览链路可用。
