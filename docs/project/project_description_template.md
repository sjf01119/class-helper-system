# 学习辅助系统项目说明模板

> 说明：本模板适用于“学习辅助系统 / 班级辅助系统 / 课程作业管理系统”类项目，可作为项目立项说明、毕业设计项目说明、系统介绍文档或交付文档基础版本。  
> 使用时可将文中的占位内容替换为你的实际项目名称、部署方式、账号规则和业务细节。

## 1. 项目概述

### 1.1 项目名称

- 项目名称：`学习辅助系统`
- 英文名称：`Class Helper System`

### 1.2 系统定位

学习辅助系统是一套面向校园教学场景的综合管理平台，围绕“班级管理、课程管理、作业管理、公告通知、学习资料共享、成绩查询与个人信息维护”等核心业务构建，服务于学校、教师与学生三类用户，帮助教学活动实现数字化、规范化和高效协同。

系统重点解决以下问题：

- 教学组织信息分散，班级、课程、学生关系难以统一管理
- 教师布置作业、发布公告、上传资料流程繁琐
- 学生获取课程信息、提交作业、查看成绩入口不集中
- 管理员对教师、学生、班级、课程等基础数据缺乏统一维护平台

### 1.3 目标用户

本系统主要面向以下三类用户：

- 管理员：负责平台基础数据维护、用户管理、班级管理、课程管理与系统公告管理
- 教师：负责教学实施，包括课程维护、班级管理、作业布置、公告发布、学习资料上传与成绩管理
- 学生：负责课程参与，包括加入班级、查看课程、提交作业、查看公告、下载资料、查询成绩与维护个人信息

### 1.4 核心价值

本系统的核心价值体现在以下几个方面：

- 提升教学管理效率：统一管理班级、课程、学生、作业和公告，减少线下重复沟通成本
- 强化教学过程协同：支持教师发布课程、布置作业、共享资料，学生在线接收并完成学习任务
- 规范数据管理流程：通过角色权限控制和数据隔离机制，保证各类用户只访问自己权限范围内的数据
- 改善学习体验：学生能够在统一入口查看课程、作业、公告、成绩与学习资料，学习路径更清晰
- 支持业务扩展：系统具备清晰的前后端分层和模块化设计，便于后续扩展考试、问答、统计分析等功能

## 2. 角色与权限

### 2.1 角色划分

系统采用基于角色的权限控制机制，主要角色包括：

- 管理员
- 教师
- 学生

### 2.2 权限说明

| 角色 | 权限范围 | 典型操作 |
|---|---|---|
| 管理员 | 平台级管理权限 | 用户管理、班级管理、课程管理、系统公告、日志查看 |
| 教师 | 教学业务权限 | 查看我的班级、维护我的课程、发布作业、上传资料、发布公告、成绩管理 |
| 学生 | 学习参与权限 | 加入班级、查看我的课程、提交作业、查看公告、下载资料、查询成绩 |

### 2.3 各角色详细权限

#### 管理员权限

- 维护教师、学生账号信息
- 分配用户角色
- 创建、编辑、删除班级
- 创建、编辑、删除课程
- 管理系统级公告
- 查看系统登录日志与操作日志
- 对全局业务数据进行查询和维护

#### 教师权限

- 查看并维护本人授课课程
- 查看本人负责的班级或教学班级
- 发布、编辑、删除作业
- 查看学生提交情况并进行批改
- 上传、管理学习资料
- 发布班级公告或教学公告
- 查看学生成绩并进行录入或维护
- 修改个人信息与头像

#### 学生权限

- 通过邀请码加入班级
- 查看本人所在班级信息
- 查看本人可见课程
- 查看课程资料与课程详情
- 提交作业并查看提交状态
- 查看班级公告和系统公告
- 查询成绩
- 修改个人信息与头像

### 2.4 权限控制原则

- 管理员拥有最高权限，可管理全平台核心业务数据
- 教师只能访问与本人有关的班级、课程、作业、公告和成绩数据
- 学生只能访问本人所在班级、本人可见课程、本人作业和本人成绩数据
- 所有涉及跨班级、跨课程的数据操作必须经过身份校验和数据归属校验

## 3. 核心功能模块

### 3.1 管理员端功能模块

#### 1. 用户管理

- 新增教师账号和学生账号
- 编辑用户基础信息
- 分配用户角色
- 启用或禁用账号
- 重置用户密码

#### 2. 班级管理

- 创建班级
- 编辑班级名称、描述、状态、邀请码等信息
- 维护班级基础信息
- 管理班级组织结构

#### 3. 课程管理

- 创建课程
- 设置课程所属教师、所属班级、描述和状态
- 维护课程生命周期信息

#### 4. 系统公告管理

- 发布系统级公告
- 编辑或删除公告
- 维护公告状态、优先级、发布时间

#### 5. 日志管理

- 查看登录日志
- 查看操作日志
- 追踪关键业务操作记录

### 3.2 教师端功能模块

#### 1. 我的班级

- 查看本人负责或管理的班级信息
- 查看班级学生列表
- 查看班级邀请码
- 了解班级公告、课程和作业情况

#### 2. 我的课程

- 查看本人授课课程列表
- 查看课程详情
- 管理课程介绍与状态
- 跳转到作业、资料、成绩等教学入口

#### 3. 作业管理

- 发布新作业
- 编辑已有作业
- 删除作业
- 查看作业提交统计
- 批改学生作业并填写评语、成绩

#### 4. 学习资料管理

- 上传课件、文档、图片等学习资料
- 编辑资料信息
- 删除资料
- 控制资料在课程或班级中的可见范围

#### 5. 公告管理

- 发布班级公告
- 编辑公告内容
- 删除公告
- 通知学生课程安排、任务提醒等信息

#### 6. 成绩管理

- 查看课程成绩数据
- 对学生成绩进行录入和维护
- 支持按课程或作业进行成绩查询

#### 7. 个人中心

- 修改个人信息
- 修改密码
- 上传头像

### 3.3 学生端功能模块

#### 1. 我的班级

- 查看所在班级信息
- 查看班级公告
- 查看班级成员基本信息
- 复制或识别班级邀请码相关信息

#### 2. 我的课程

- 查看当前可学习课程
- 查看课程描述、更新时间、学习入口
- 跳转到作业中心、学习资料、成绩查询

#### 3. 作业中心

- 查看待提交作业和已提交作业
- 在线填写文字内容
- 上传附件提交作业
- 查看提交状态、截止时间和批改结果

#### 4. 学习资料

- 浏览课程相关学习资料
- 预览或下载资料文件

#### 5. 公告通知

- 查看系统公告
- 查看班级公告
- 查看公告发布时间与详细内容

#### 6. 成绩查询

- 查看各课程成绩
- 查看作业得分与教师反馈

#### 7. 个人中心

- 修改个人基础信息
- 修改密码
- 上传头像

## 4. 关键业务规则

### 4.1 班主任管理规则

- 一个班级应绑定一个明确的班主任或负责人
- 班主任负责查看和管理自己负责的班级信息
- 教师是否显示“我的班级”菜单，可根据是否担任班主任或是否具备班级管理关系动态控制
- 非班主任教师不得查看不属于自己的班级详情

### 4.2 课程管理规则

- 每门课程必须关联授课教师
- 每门课程必须关联所属班级
- 教师只能查看和维护自己授课的课程
- 学生只能查看本人所在班级可见的课程
- 课程创建、编辑、状态切换时应同步维护更新时间

### 4.3 作业管理规则

- 教师只能在自己授课课程下发布作业
- 作业必须归属于某一课程
- 作业应包含标题、内容、截止时间、总分等基本信息
- 学生提交作业时，至少应提供文字内容或附件中的一种
- 作业过截止时间后，默认不允许继续提交，除非系统另有补交规则
- 教师批改后，学生可查看得分与评语

### 4.4 公告管理规则

- 公告分为系统公告与班级公告
- 系统公告由管理员统一发布
- 班级公告由教师发布，且仅对对应班级学生可见
- 学生不可发布公告

### 4.5 文件管理规则

- 上传文件需校验文件类型、大小和安全性
- 学习资料、作业附件、头像文件应分目录存储
- 文件上传成功后仅保存访问地址或存储路径到数据库
- 学生和教师只能访问自己有权限查看的文件
- 删除业务数据时，应同步考虑文件引用关系，避免产生无效链接

### 4.6 个人信息与头像规则

- 用户只能修改自己的个人资料
- 用户只能上传和更新自己的头像
- 头像格式通常限制为 `JPG / JPEG / PNG`
- 头像上传成功后应立即更新前端展示状态

## 5. 技术实现说明

### 5.1 系统架构

本系统采用前后端分离架构，整体分为：

- 前端：负责页面展示、交互逻辑、表单校验与路由控制
- 后端：负责业务处理、权限控制、数据访问、文件上传与接口输出
- 数据库：负责存储用户、角色、班级、课程、作业、公告、资料、日志等核心业务数据
- 文件存储：负责保存学习资料附件、作业附件、头像等静态文件

### 5.2 数据库设计说明

数据库通常围绕以下核心实体展开：

| 数据表 | 作用说明 |
|---|---|
| `sys_user` | 存储系统用户基础信息 |
| `sys_role` | 存储角色定义 |
| `sys_user_role` | 存储用户与角色关联关系 |
| `sys_class` | 存储班级信息 |
| `sys_course` | 存储课程信息 |
| `assignment` | 存储作业信息 |
| `assignment_submission` | 存储学生作业提交记录 |
| `learning_resource` | 存储学习资料信息 |
| `sys_announcement` | 存储公告信息 |
| `sys_login_log` | 存储登录日志 |
| `sys_operation_log` | 存储操作日志 |

### 5.3 文件存储实现

系统文件存储可采用以下方式：

- 本地文件存储：适用于开发环境或小规模部署
- 对象存储服务：适用于正式环境，如 OSS、COS、MinIO 等

文件一般按业务类型独立目录管理，例如：

- `uploads/avatars/`：头像文件
- `uploads/materials/`：学习资料文件
- `uploads/homework/`：作业附件

数据库中通常只保存：

- 文件访问 URL
- 文件相对路径
- 文件名称
- 文件大小
- 上传人信息

### 5.4 权限控制逻辑

系统权限控制一般采用“认证 + 授权 + 数据隔离”三层机制：

#### 1. 认证

- 用户通过用户名和密码登录
- 登录成功后生成令牌，如 JWT
- 前端后续请求通过请求头携带令牌

#### 2. 授权

- 后端根据当前登录用户角色判断其可访问的接口
- 管理员、教师、学生访问不同模块时进行角色校验

#### 3. 数据隔离

- 教师只能操作自己的课程、作业和班级
- 学生只能查看自己的课程、作业和成绩
- 非法访问时后端返回无权限提示

### 5.5 前后端交互说明

- 前端通过统一请求封装调用后端 RESTful 接口
- 后端统一返回标准响应结构，便于前端解析成功与失败状态
- 页面数据通过表格、卡片、弹窗、详情页等形式展示
- 关键操作包括新增、编辑、删除、上传、提交、批改、查询等

## 6. 使用流程

以下以“教师发布课程 → 学生加入 → 作业提交 → 教师批改”为例，说明系统完整业务链路。

### 6.1 教师发布课程

1. 管理员先创建班级、教师和学生账号
2. 管理员创建课程，并将课程分配给指定教师和班级
3. 教师登录系统后，在“我的课程”中查看已分配课程
4. 教师完善课程描述、上传资料或设置相关教学内容

### 6.2 学生加入班级

1. 学生注册或由管理员创建学生账号
2. 学生登录系统后进入“我的班级”或加入班级页面
3. 学生输入班级邀请码加入对应班级
4. 系统验证邀请码有效性后，将学生与班级建立关联
5. 学生加入班级后，即可看到所在班级的课程、公告与作业

### 6.3 教师布置作业

1. 教师进入“作业管理”
2. 选择所属课程并填写作业标题、内容、截止时间、总分等信息
3. 如有需要，教师上传作业附件
4. 提交后，系统生成作业记录并关联到课程与班级
5. 学生在自己的作业中心即可看到该作业

### 6.4 学生提交作业

1. 学生进入“作业中心”
2. 查看待提交作业列表
3. 打开作业详情，填写文字答案或上传附件
4. 提交后，系统生成学生提交记录
5. 学生可在“已提交作业”中查看提交时间、状态和后续批改结果

### 6.5 教师批改作业

1. 教师进入“作业管理”
2. 查看某份作业的学生提交列表
3. 打开学生提交详情，查看文本内容和附件
4. 教师填写成绩和评语后提交批改结果
5. 系统保存批改记录，并将结果同步到学生成绩查询或作业结果页面

### 6.6 学生查看结果

1. 学生在作业中心查看该作业的批改状态
2. 学生查看教师给出的得分和评语
3. 如系统集成成绩模块，相关分数也可同步在成绩查询中展示
