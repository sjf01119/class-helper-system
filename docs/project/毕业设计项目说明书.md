## 摘要

随着高校教学数字化转型持续推进，传统教学管理中存在课程信息分散、作业流程不闭环、师生沟通路径长、教学数据难追溯等问题。为解决上述问题，本文结合当前项目仓库 `class-helper-system` 的实际实现，设计并实现了一套面向管理员、教师、学生三类角色的学习辅助系统。系统采用前后端分离架构，围绕“班级管理、课程管理、作业管理、公告通知、学习资料管理、成绩管理、日志审计”七类核心业务构建统一平台。

在技术实现方面，后端基于 Spring Boot 3.2.2、Spring Security、JWT 与 MyBatis-Plus 完成认证鉴权、权限控制与业务服务开发；前端基于 Vue 3、TypeScript、Vite、Element Plus 实现三端角色化界面与交互流程；数据层使用 MySQL，系统支持本地文件上传与阿里云 OSS 扩展存储。通过对关键业务链路（课程发布、作业布置、学生提交、教师批改、成绩反馈）进行功能验证，系统具备较好的可用性、可维护性和扩展性，能够满足课堂教学协同场景下的日常管理需求。

关键词：学习辅助系统；Spring Boot；Vue 3；JWT；MyBatis-Plus

## Abstract

With the continuous digital transformation of higher education, traditional teaching management still faces issues such as fragmented course information, incomplete assignment workflows, inefficient communication, and limited data traceability. To address these challenges, this project designs and implements a Java Web-based Class Helper System for three roles: administrators, teachers, and students, based on the real codebase `class-helper-system`.

The system adopts a front-end and back-end separation architecture and covers core modules including class management, course management, assignment management, announcements, learning resources, grade management, and operation logging. The backend is built with Spring Boot 3.2.2, Spring Security, JWT, and MyBatis-Plus, while the frontend is developed with Vue 3, TypeScript, Vite, and Element Plus. MySQL is used for persistent storage, and the system supports both local file upload and OSS-based file storage extension. Functional verification on key workflows demonstrates that the system has good usability, maintainability, and extensibility for practical teaching collaboration scenarios.

Key Words: Class Helper System; Spring Boot; Vue 3; JWT; MyBatis-Plus

## 1 绪论

### 1.1 研究背景

当前高校教学管理逐步从线下转向线上协同，但在多数教学场景中仍存在多个系统割裂使用、作业提交渠道不统一、公告触达效率低、教学过程数据沉淀不足等问题。尤其在班级规模较大、课程数量较多的情况下，教师在作业布置、批改统计、资料共享等环节压力显著增加，学生在课程跟进与学习反馈方面也缺乏统一入口。

### 1.2 研究目的与意义

本项目旨在构建一套覆盖教学核心流程的学习辅助系统，实现管理员、教师、学生三方在同一平台下的协同操作。通过系统建设，达到以下目标：

- 建立统一的数据入口，规范班级、课程、用户与作业等基础信息管理。
- 建立作业全流程闭环，提高教学执行效率与结果可追踪性。
- 建立角色化权限控制，保障系统安全与数据边界。
- 为后续扩展统计分析、智能推荐和教学质量评估提供数据基础。

### 1.3 国内外发展现状简述

国内外教学平台普遍具备课程发布、作业管理和学习资料共享能力，但在中小规模院校与课程团队场景中，平台常见问题包括：功能过重导致使用门槛高、二次定制成本高、角色权限粒度不匹配教学实际流程。本项目聚焦“课程与班级协同”核心场景，以轻量化实现与可扩展架构为导向，强调工程落地与可维护性。

### 1.4 研究方法

项目采用文献调研、需求分析、模块化设计、分层实现和功能验证相结合的方法。技术路线遵循“需求先行、架构分层、接口驱动、逐步验证”的工程思路。

## 2 系统分析

### 2.1 可行性分析

#### 2.1.1 技术可行性

当前项目已实现完整前后端基础能力：后端具备认证鉴权、角色权限、日志审计与核心业务模块；前端具备三角色路由控制与页面功能。所采用技术栈成熟，社区生态完善，满足毕业设计工程实现需求。

#### 2.1.2 经济可行性

开发框架与工具以开源组件为主，硬件与软件成本较低。系统可在普通开发机与常见云服务器部署，符合学生毕业设计资源条件。

#### 2.1.3 应用可行性

系统功能与教学管理业务高度匹配，能够直接服务课程组织、作业管理和学习资料共享等教学高频场景，具有现实应用基础。

### 2.2 角色与需求分析

#### 2.2.1 角色划分

- 管理员：平台级维护与监管。
- 教师：教学组织与任务发布执行。
- 学生：学习参与与任务完成反馈。

#### 2.2.2 功能需求

- 管理员：用户、班级、课程、公告、日志、看板管理。
- 教师：我的班级、我的课程、作业管理、学习资料、公告管理、成绩管理。
- 学生：加入班级、查看课程、提交作业、查看公告、查看资料、成绩查询。

### 2.3 关键业务流程

典型流程为“课程配置 → 作业发布 → 学生提交 → 教师批改 → 成绩反馈”。该流程在当前系统中已具备完整入口与数据落地路径，构成教学任务执行闭环。

## 3 总体设计

### 3.1 系统设计目标

- 功能完整：覆盖教学协同核心业务。
- 权限清晰：不同角色访问边界明确。
- 结构清晰：前后端职责分离、模块边界明确。
- 易于扩展：支持后续功能迭代和部署升级。

### 3.2 系统架构设计

系统采用前后端分离架构：

- 表现层：Vue 3 页面与交互逻辑。
- 接口层：Spring Boot 控制器提供 REST API。
- 业务层：Service 处理业务规则与事务逻辑。
- 持久层：Mapper + MyBatis-Plus 完成数据访问。
- 基础层：JWT、安全过滤、日志切面、文件存储能力。

### 3.3 数据设计概述

核心数据实体包括用户、角色、用户角色关系、班级、课程、作业、作业提交、公告、学习资料、登录日志、操作日志等。数据模型支持按“角色-班级-课程”维度进行业务关联与权限收敛。

## 4 详细设计与实现

### 4.1 后端实现

#### 4.1.1 认证与授权

登录后由后端签发 JWT，前端请求携带令牌访问受保护接口。接口层通过角色注解与安全上下文进行双重鉴权，实现“接口可访问 + 数据可访问”的组合控制。

#### 4.1.2 业务模块

- 用户与角色模块：支持账号管理、角色分配、个人信息维护。
- 班级与课程模块：支持班级维护、课程配置、师生课程关联。
- 作业模块：支持作业发布、附件上传、提交记录、评分反馈。
- 公告与资料模块：支持公告发布与学习资源管理。
- 日志模块：记录登录与关键操作，支持审计追踪。

### 4.2 前端实现

前端按角色路由拆分为管理员端、教师端、学生端。通过路由守卫校验登录态与角色范围，避免串后台访问。页面层围绕“列表 + 详情 + 表单 + 操作反馈”交互模式构建，兼顾操作效率与学习成本。

### 4.3 关键实现特点

- 三角色菜单与路由隔离。
- 作业附件上传、下载与访问链路完整。
- 班级邀请码加入机制可用。
- 日志查询与数据看板具备可视化支撑。

## 5 系统测试

### 5.1 测试目标

验证系统在认证授权、角色隔离、核心业务流程和接口稳定性方面是否满足预期。

### 5.2 测试方法

- 接口测试：依据 `backend/API-TEST.md` 执行登录与角色接口验证。
- 功能测试：按角色执行主要业务链路操作。
- 异常测试：验证越权访问、无效参数、无令牌请求处理结果。

### 5.3 测试结果分析

测试结果表明：系统可完成三角色核心业务；权限控制有效阻断越权行为；作业闭环流程可正常执行；日志模块可追踪关键操作。整体满足课程项目与毕业设计演示要求。

## 6 部署与运行维护

### 6.1 运行环境

- JDK 17+
- MySQL 8+
- Node.js 18+
- Maven 3.8+

### 6.2 本地部署步骤

1. 初始化数据库并导入 SQL 脚本。  
2. 在 `backend` 目录执行 `mvn spring-boot:run` 启动后端。  
3. 在 `frontend` 目录执行 `npm install && npm run dev` 启动前端。  
4. 通过浏览器访问前端地址并联调接口。  

### 6.3 生产部署建议

- 前端使用 Nginx 托管静态资源并配置路由回退。
- 后端使用 Jar 包运行，推荐 `prod` 配置。
- 核心配置通过环境变量管理（数据库、JWT、CORS、OSS）。
- 定期执行数据库备份并保留回滚方案。

## 7 结论与展望

### 7.1 结论

本文完成了基于 Java Web 的学习辅助系统设计与实现工作，构建了覆盖管理员、教师、学生三角色的教学协同平台。系统在课程、班级、作业、公告、资料、成绩和日志等方面实现了较完整的业务支持，验证了前后端分离架构在教学场景中的可行性与实用性。

### 7.2 不足与改进方向

- 自动化测试覆盖仍需加强。
- 权限粒度可从角色级扩展到菜单与按钮级。
- 数据看板可增加更细粒度教学行为分析。
- 可进一步引入智能问答与学习建议功能。
